Validate GitHub issue is ready for implementation.

Input: issue_number

Steps:
1. Fetch issue: gh issue view <number> --json title,body,labels,state
2. Validate state:
   - Check: issue is OPEN (not CLOSED or DRAFT)
   - Check: has "ready-for-work" label
   - Check: does NOT already have an active implementation branch for this issue (git branch -a | grep "issue-<number>")
   - Check: caller supplied a `branch` input (validated by the invoking orchestrator; a missing branch is an ABORT there, not a state this skill fetches from GitHub)
3. Analyze issue to determine commit type:
   - feat: if new feature/enhancement
   - fix: if bug fix
   - refactor: if code improvement without behavior change
   - docs: if documentation only
   - test: if test-only change
4. Output success or failure with clear reasons

Success output:
- Issue title
- Issue number
- Issue state: OPEN
- Has ready-for-work label: YES
- Commit type: <type>
- Ready to proceed: YES

Failure output:
- Issue number
- Reason (missing label, wrong state, etc.)
- Recommended action to fix
